About This Project

The India Boundary Corrector provides accurate representations of India's boundaries in web maps, correcting discrepancies in OpenStreetMap and Natural Earth data. These examples demonstrate how to integrate the corrections into MapLibre GL JS applications.

Quick Start

Install the packages:

npm install @india-boundary-corrector/maplibre maplibre-gl pmtiles

Basic usage:

import maplibregl from 'maplibre-gl';
import { Protocol } from 'pmtiles';
import { addBoundaryCorrector } from '@india-boundary-corrector/maplibre';

// Register PMTiles protocol
const protocol = new Protocol();
maplibregl.addProtocol('pmtiles', protocol.tile);

const map = new maplibregl.Map({
  container: 'map',
  style: {
    version: 8,
    sources: {
      'carto-dark': {
        type: 'raster',
        tiles: ['https://a.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}.png'],
        tileSize: 256
      }
    },
    layers: [{ id: 'base', type: 'raster', source: 'carto-dark' }]
  },
  center: [78.9629, 20.5937],
  zoom: 4,
});

map.on('load', () => {
  // Auto-detects tile layers and adds corrections
  const corrector = addBoundaryCorrector(map);
  
  // Or with explicit config:
  // const corrector = addBoundaryCorrector(map, {
  //   layerConfig: 'osm-carto-dark'
  // });
});

How It Works

The boundary correction system uses a three-layer approach:

  1. Base Raster Layer: Renders the original tile source (e.g., OSM Carto Dark)
  2. Delete Layers: Draws lines matching the background color to mask incorrect boundaries
  3. Add Layers: Draws the correct India boundaries in the appropriate style

Different correction sources are automatically used based on zoom level: Natural Earth data for lower zooms (< 5) and OpenStreetMap data for higher zooms (‚â• 5).
